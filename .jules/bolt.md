## 2024-05-18 - Monte Carlo Deck Sampling Optimization
**Learning:** `random.shuffle` over the entire array inside a hot loop is a common but extremely inefficient pattern when only a small portion of the array is needed. In `EquitySolver`, shuffling a 45-card deck for each simulation took significantly longer than just sampling the needed cards. `random.sample` is much faster for drawing hands since it avoids shuffling the entire deck array.
**Action:** When picking a small subset of elements randomly in a performance-critical loop, use `random.sample` instead of shuffling the whole array and taking a slice.

## 2024-05-18 - Nested Loop Set Memberships in Range-vs-Range Monte Carlo
**Learning:** During Monte Carlo evaluations for `compute_range_vs_range_equity` where each simulation evaluates dozens of hero hands against a random villain hand and board, doing linear lookups like `if h_hand[0] in v_hand` or `if h_hand[0] in sampled_board` inside the innermost hot loop absolutely demolishes performance (taking ~3.8s per 100k inner loops). Constructing a `forbidden` set outside the inner loop (`forbidden = {v_hand[0], v_hand[1]}; forbidden.update(sampled_board)`) and using fast set lookups (`if h_hand[0] in forbidden`) cuts the overhead by almost 4x (~0.95s per 100k loops). Also, using `random.sample` over dynamically filtered lists instead of repeated `while len(...) < needed: random.choice()` with element membership checks is about 1.3x faster for drawing boards.
**Action:** Always replace multiple list/tuple `in` checks inside tight Python loops with a pre-computed `set`, even if the set has to be recomputed on every outer iteration. Additionally, avoid `while` loops for random sampling in favor of `random.sample` over filtered lists.
