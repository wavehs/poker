## 2024-05-18 - Monte Carlo Deck Sampling Optimization
**Learning:** `random.shuffle` over the entire array inside a hot loop is a common but extremely inefficient pattern when only a small portion of the array is needed. In `EquitySolver`, shuffling a 45-card deck for each simulation took significantly longer than just sampling the needed cards. `random.sample` is much faster for drawing hands since it avoids shuffling the entire deck array.
**Action:** When picking a small subset of elements randomly in a performance-critical loop, use `random.sample` instead of shuffling the whole array and taking a slice.
## 2025-02-15 - Unrolled Combinations & Allocation-Free Logic in Evaluator
**Learning:** The BuiltinEvaluator originally suffered heavily from object allocation overhead inside tight Monte Carlo loops (e.g. creating dictionaries to count rank occurrences, calling `.values()`, `.items()`, using `set()` to detect straights). Furthermore, calling `itertools.combinations(cards, 5)` in the 6/7 card evaluation step allocates a generator and creates tuple objects dynamically for each combination, drastically slowing down evaluation.
**Action:** By completely removing object allocations (lists, dictionaries, sets) from `_evaluate_five_int`, applying a manual sorting network, replacing exponentiation (`15**4`) with constants (`50625`), and explicitly unrolling `itertools.combinations` for the 6-card and 7-card evaluator, the BuiltinEvaluator achieves ~500-600% speedup. In Python, manually unrolling loops and eliminating object allocations is a massive performance win for hot paths.
